xml基础

xml

  1. 概念:
    1. Extensible Markup Language 可扩展标记语言.
      • 可扩展: xml标签都是自定义的,例如,.
    2. 功能
      • 存储数据:
        1. 可用作配置文件.
        2. 可用于在网络中传输.
    3. xml与html的区别:
      • 他俩的亲爹都是w3c(万维网联盟),XML 是 W3C 的推荐标准.xml晚于html产生,因为html由于浏览器的恶性竞争(浏览器解析能力不同)导致html语法不那么严格也可以达到相同的效果.
      • xml始终争不过html,所以它改为与properties(只能存键值对)竞争,例如name = zhang, gender = nan 和 name = li, gender = nv 在这里我凭什么不可以认为zhang的性别是nv呢?而xml可以通过标签包裹用户信息,例如<user id = '1'><name>zhang</name> <gender>nan</gender></user>,这样子可读性更高,而且容易理解.
      • 区别1: xml的标签都是自定义的,html标签都是预定义的.
      • 区别2: xml的语法严格,html的语法松散,很难写错.
      • xml是存储数据的,html是展示数据的(将数据内容展示到页面上).

  2. 语法

    1. 基本语法:
      1. xml文档的后缀名: .xml.
      2. 第一行必须是文档声明,即<?xml version = '1.0' ?>,顶行写,必须在第一行,不过我在谷歌浏览器,不写文档声明也没有报错,但是还是写上最好.
      3. 一个xml文档有且只有一个根标签,如例子中的<users>.
      4. 属性值必须使用引号引起来,单双引号都可以.
      5. 标签必须为一对,即有开始就必须有结束,如<abc></abc>,或者为自闭合标签,如<abc/>.
      6. xml标签区分大小写,如<abc></ABC>会报错.

    2. 快速入门:将下列例子存放在xml文本文件中再直接用浏览器打开,没报错就说明写对了 .

    3. 组成部分:

      1. 文档声明
        1. 格式: <?xml 属性列表 ?>
        2. 注意:在<?xml 属性列表 ?>中,?和<>和xml三者之间不能有空格,有空格就会报错.
        3. 属性列表:
          1. version: 版本号,只写1.0,写1.1会出问题,它不向下兼容,没有1.2版本,而且,这个version是必须写的,不写会报错.
          2. encoding: 编码方式,告知解析引擎当前文档使用的字符集,默认是UTF-8,不写也可以,一般就写UTF-8就行,写其他的可能会出错.
          3. standalone(了解): 是否独立.
            1. 有两个值yes和no.
              • yes表示不依赖其他文件.
              • no表示依赖其他文件.
              • 不过这个我们一般不管他.
      2. 指令(了解)
        1. 可以结合css使用,现在不用,现在xml只用来存储数据.
      3. 标签
        1. 标签名称是自定义的.
        2. 命名规则超链接: xml命名规则.
      4. 属性
        1. id属性值唯一.
      5. 文本内容
        1. CDATA区: 在该区域的数据会被原样展示,因为有的符号需要转义xml才能识别,有了这个就不需要转义了.
        2. 格式: <![CDATA[语句]]>
    4. 约束
      1. 谁编写xml?—用户写,即软件的使用者(就是程序猿啦)
      2. 谁解析xml?—被使用的软件(半成品软件,理解为框架),但是这里有一个问题,就是我的这个框架怎么知道程序猿写的标签是要干嘛的呢?所以这里就需要一个约束来规定一下,什么标签是用来干嘛的,这就是该软件提供的说明文档,也叫约束文档,用于说明xml文档的书写规则.
      3. 约束概念: 规定xml文档的书写规则.
        1. 作为框架的使用者(程序猿):
          1. 能够在xml中引入约束文档.
          2. 能够简单的读懂约束文档.
      4. 分类:
        1. DTD: 简单的约束技术,不过缺陷蛮多.
          1. 引入dtd文档到xml文档中
            1. 内部dtd: 将约束的规则定义到xml文档中(了解).
            2. 外部dtd: 将约束的规则定义在外部的文件中.
              1. 本地引入: <!DOCTYPE 根标签 SYSTEM "dtd文件的位置">.
              2. 网络引入: <!DOCTYPE 根标签 PUBLIC "dtd文件的名字" "dtd文件的位置(url位置)">.
          2. 缺陷: dtd中的约束为sex必须字符串,我希望的是只有男和女,但是如果xml文档中写了个不知道也是不会报错可以正常运行的,这个就不合规范比较随意,不是我们想要的.
          • DTD工具和例子.
        2. Schema: 较复杂的约束技术,可以限制内容.
          • Schema工具和例子.

  3. 解析: 操作xml文档,将文档中的数据读取到内存中.

    1. 操作xml文档:
      1. 解析(读取):将文档中的数据读取到内存中.
      2. 写入: 将内存中的数据保存到文档中,持久化的存储.
    2. 解析xml的方式:
      1. DOM:将标记语言文档一次性加载进内存,在内存中形成一颗DOM树结构
        1. 优点: 操作方便,可以对文档进行CRUD的所有操作.
        2. 缺点: 由于是一次加载所有的进内存,所以比较占内存.
      2. SAX:逐行读取,而且是读到下一行,上一行就释放了,所以在内存中一直都只有一行代码,同时这个是基于事件驱动的.
        1. 优点: 不占内存.
        2. 只能读取,不能增删改.
    3. xml常见解析器:
      1. JAXP(sun公司提供的):支持DOM和SAX两种思想,性能很低,不用.
      2. DOM4J:一款非常优秀的解析器,基于DOM思想.
      3. Jsoup:本来是用来解析html的,是一款Java 的HTML解析器，可直接解析某个URL地址、HTML文本内容。它提供了一套非常省力的API，可通过DOM，CSS以及类似于jQuery的操作方法来取出和操作数据。
        1. Jsoup工具.
      4. PULL:安卓操作系统内置的解析器,是SAX方式的.

      2. 对象的使用:

        1. Jsoup: 工具类, 可以解析html和xml文档,返回Document对象.
          1. 注意其中的parse()方法,用于解析html和xml文档,返回Document对象.
            1. (重点)`public static Document parse​(File in,
              String charsetName)`:解析html或xml文件的.
            2. public static Document parse​(String html): 解析xml或html字符串.
            3. `public static Document parse​(URL url,
              int timeoutMillis)`: 通过网络路径获取指定的html或xml的文档对象.
          2. Jsoup对象案例代码.
        2. Document(继承自Element): 文档对象,代表内存总的dom树.
          1. 主要用于获取Element对象(Element是Elements的父类)
            1. public Elements getElementsByTag​(String tagName): 根据标签名称获取元素对象的集合.
            2. public Elements getElementsByAttribute​(String key): 根据属性名称来获取元素对象集合.
            3. `public Elements getElementsByAttributeValue​(String key,
              String value)`: 根据对应的属性名和值来获取元素对象集合.
            4. public Element getElementById​(String id): 根据id值获取唯一的Element对象.
          2. document对象演示案例代码.
        3. Elements: 元素Element对象的集合,可以当作ArrayList来使用.
        4. Element: 元素对象,可以获取元素名称,属性等.
          1. 获取子元素对象
            1. public Elements getElementsByTag​(String tagName): 根据标签名称获取元素对象的集合.
            2. public Elements getElementsByAttribute​(String key): 根据属性名称来获取元素对象集合.
            3. `public Elements getElementsByAttributeValue​(String key,
              String value)`: 根据对应的属性名和值来获取元素对象集合.
            4. public Element getElementById​(String id): 根据id值获取唯一的Element对象.
          2. 获取属性值
            1. public String attr(String attributeKey): 根据属性名称(不区分大小写)获取属性值.
          3. 获取文本内容
            1. public String text​(): 获取所有子标签的纯文本内容.
            2. public String html​(): 获取标签体的所有文本内容,包括子标签的字符串内容.
          4. Element对象案例代码.
        5. 节点对象: 上述几个对象的爹,是Document和Element的父类.
